## Project Overview

### Description

Authentication bypass was achieved through careful manipulation of the login mechanism. Initial enumeration revealed several interesting open ports on the target. The database credentials were hardcoded in the application source code. Unconstrained delegation was enabled on the compromised machine.

## Features

### Key Features

- **impacket**: The target system was identified as running a vulnerable version of the service.
- **hashcat**: Weak file permissions allowed modification of critical system files.
- **gobuster**: The kernel version was outdated and vulnerable to a publicly known exploit.
- **john**: The backup files contained sensitive information that should not have been accessible.

## Installation

### Requirements
